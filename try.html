SunnyDreams Hotel Interface

It began as a small front-end experiment—designing a clean travel landing page—but quickly grew into something more intentional. SunnyDreams became an opportunity to explore how structure, layout, and dynamic content can work together to create a smooth and engaging browsing experience.

Rather than simply placing elements on a page, the focus shifted toward how each section communicates with the user. The navigation, featured hotel listings, and client feedback sections were designed to guide attention naturally while maintaining a clean and visually balanced interface.

As development progressed, the project became an exercise in thinking beyond static design. JavaScript was introduced to render content dynamically, while components were organized in a scalable way to support future growth. Every decision focused not only on appearance, but also on behavior and adaptability.

Trip to Dubai

What started as a mobile-based promotional concept for Nestlé IDEAL’s “Trip to Dubai” campaign quickly developed into a deeper exploration of how USSD technology can deliver large-scale, interactive brand experiences.

Rather than relying on traditional digital platforms, the focus shifted toward accessibility and simplicity. The interaction flow was designed so users could easily enter unique product codes, receive instant feedback, and remain engaged throughout the campaign—all from basic mobile devices.

As the system evolved, it became more than just a promotional tool. Structuring user sessions, response handling, and reward flows required careful planning to ensure the experience stayed responsive, reliable, and scalable. Beneath the surface, it’s a system designed to support high engagement while maintaining a smooth user experience.

Temperature Conversion App

This temperature conversion application was designed to provide users with a fast and straightforward way to switch between Celsius and Fahrenheit through a clean, minimal interface.

The focus extended beyond basic functionality to how users interact with simple tools. Attention was given to creating clear inputs, responsive feedback, and an experience that feels intuitive and effortless.

During development, JavaScript played a key role in handling real-time calculations, validating user input, and dynamically updating the interface based on user selections. Over time, the project became more than just a conversion tool—it evolved into an exercise in improving UI clarity, structuring workflows, and strengthening the relationship between HTML, CSS, and JavaScript in building effective web applications.
